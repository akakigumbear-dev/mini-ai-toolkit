  ---
  Architecture at a glance

  3 deployable units in a monorepo with a shared types package:

  ┌──────────────────┬─────────────────────────────────┬───────────────────┐
  │     Service      │              Role               │      Exposes      │
  ├──────────────────┼─────────────────────────────────┼───────────────────┤
  │ API Gateway      │ REST API, WebSocket server,     │ HTTP :3001, WS    │
  │ (NestJS)         │ queue producer/consumer         │ :3001             │
  ├──────────────────┼─────────────────────────────────┼───────────────────┤
  │ Generation       │ AMQP consumer, prompt           │ No HTTP — pure    │
  │ Worker (NestJS)  │ enhancement, AI generation      │ queue consumer    │
  ├──────────────────┼─────────────────────────────────┼───────────────────┤
  │ Web (Next.js)    │ UI — prompt form, job feed,     │ HTTP :3000        │
  │                  │ gallery, history                │                   │
  └──────────────────┴─────────────────────────────────┴───────────────────┘

  Infrastructure: PostgreSQL + RabbitMQ + Docker Compose

  Core async flow

  User → POST /api/jobs → DB (pending) → RabbitMQ (queued) → 201 returned
  immediately
                                                ↓
                                       Worker picks up job
                                       Enhances prompt → Calls Pollinations (10s
   timeout)
                                                ↓
                                       Publishes status to status_updates queue
                                                ↓
                                API Gateway consumes → updates DB → broadcasts
  via WebSocket

  Key decisions

  - Retry creates a new job row — preserves complete history, attempts counter
  tracks lineage
  - Cancel via DB flag — worker checks DB before processing; avoids complex
  cancel-message routing
  - Manual ACK + prefetch=1 — at-least-once delivery with backpressure
  - Provider abstraction — strategy pattern enables Pollinations → mock fallback
  - Single jobs table — right-sized, no premature normalization
  - Fanout exchange for status — decouples worker from consumers

  (architecture + async sequence). Ready to move to implementation whenever you
  are.
